如何在文章中引入图片

先说一下问题，在about 目录或者在这个一个层级新建一个目录，往里边放图片是可以生成的。但是在_post 里放图片，最后生成的目录里没有图片，只有一个html页面。 想改源码，但是找找没有找到。后来找了下既然官方都没有支持，那肯定也是不好改或者有它的想法，考虑到时间上的投入，就自己先按照自己的思路改下

1

首先，现在网上找了下，全是一样的不知道谁是原创的文章，要安装一个插件

npm install hexo-asset-image –save

不好使，卸载

npm uninstall hexo-asset-image

2

官网看看 https://hexo.io/zh-cn/docs/asset-folders

两种解决方式。 第一种很简单，在about相同层级新建images 目录，然后所有文章都引用里边的图片，缺点就是需要手动的向images 目录里放文件。

第二种使用标签插件，，，，，看都不想看。看完了发现是比较难受的方式，就是md 不支持的语法，必须要生成页面后才能看到效果

3

最后的解决办法，1 不用手动拷贝图片，2 就用md 的语法，这样md编辑器里也可以看到效果。

其实还是第一种解决办法，只是第一种的插件在hexo 3 里过期了，需用用另个一github上的

解决步骤：

1

修改配置文件_config.yml 为

post_asset_folder: true

2

我使用的是typora . 文件-> 偏好设置

3

npm install https://github.com/CodeFalling/hexo-asset-image --save

如何设置背景图片

打开文档下themes\next\source\css\ _custom\custom.styl

// 设置背景图片
body {
    background:url(/imgs/16pic_5550571_b.jpg);
    background-repeat: repeat;
    background-attachment:fixed;
    background-position:50% 50%;
    background-size:200px;
}

其他一些样式也可以在这文件里设置

关于炉石

发表于 2019-01-01 | 更新于 2019-08-06 | 分类于 仓库 | 评论数：
本文字数： | 阅读时长 ≈

这个游戏其实从一出来我就接触了。断断续续玩到了现在。
1 充钱
首先说这个游戏的核心是什么，那就是充钱。游戏里边的卡对于大多数人来说性价比不高。以前也听说过那种纸的卡片也挺贵的，所以怎么说呢，虽然是合理的但是还是接受不了。就拿现在最新的更新包来说吧，488块. 买的只是一个更新包。 而在别的平台上可以买一个不错的游戏了。而我更愿意买一件中意的衣服或鞋子。
2 随机
你花钱了还是不行的，你要花很多的钱才可以。 因为开卡是随机的。可能你开到的卡能不能用上全看脸。
就算你凑够了一个卡组，能不能赢也是有很大随机性的。因为可操作东西不多。一个自己发牌员的随机和对方发牌员的随机。你胡的时候对面刚好针对你。
所以玩这个游戏，千万不要认真。重要是体验那种赌博的快感。没错就是赌博。简直和那种赌博差不多，都是由发牌员发牌，然后比大小。
对手职业不确定，对手的牌库类型不确定，牌序不确定，还有很多卡牌的效果也是随机的。所以这个游戏的娱乐性是显而易见的。 当然也不能说没有技术可言，就是赌就行了。
3 积累
如果你是一个坚持的人，可以每天完成任务积累金币，那也还好。但我不是，我也不想。我曾经这么试过积累金币的过程太变态了。我不是一个能坚持下去的人、
4 封闭游戏
没有社交功能，游戏也不温不火，我的朋友圈没有人玩，就我自己玩。

总结，
这个游戏真的不能认真，不然能气死自己。还记得我刚开始玩的心态是，不管自己的还是对面随机的牌不错了都会觉得不可思议。所以我有一次的弃坑了。这几天一直在想要不要488买一下最新的扩展包，因为玩了这么久，还是没有上传说。最后还是算了，因为实在不想自娱自乐地玩游戏了。而且我一输就心态崩溃，尤其是我现在的时间这么宝贵，所以不能浪费钱又浪费时间。我还是多把时间放到学习、设计和交朋友上。
